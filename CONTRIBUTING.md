## 📖 项目简介

这是一本**中文社区驱动**的 GexBot 交易实战手册，专为做 SPX 0DTE（末日期权）的日内交易员设计。

我们的目标是让每一个中文交易员都能看懂 GEX 数据，做好末日期权交易。

## 🎯 如何贡献

### 方式 1：通过 GitBook 编辑（推荐新手）

**零门槛参与！** 你完全不需要会编程：

1. 访问 [GitBook 在线版](https://kangs-personal-organization-1.gitbook.io/spx0dtegexguide)
2. 点击页面底部的「在 GitHub 上编辑此页面」链接
3. 在 GitHub 上编辑内容
4. 提交 Pull Request

### 方式 2：通过 GitHub 直接贡献

1. **Fork 本仓库**
   ```bash
   # 在 GitHub 上点击 Fork 按钮
   ```

2. **克隆到本地**
   ```bash
   git clone https://github.com/YOUR_USERNAME/spx0dtegexguide.git
   cd spx0dtegexguide
   ```

3. **创建分支**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **进行修改**
   - 编辑 Markdown 文件
   - 添加你的交易笔记
   - 修复错误或改进内容

5. **提交更改**
   ```bash
   git add .
   git commit -m "feat: 添加你的功能描述"
   git push origin feature/your-feature-name
   ```

6. **创建 Pull Request**
   - 在 GitHub 上创建 PR
   - 描述你的更改内容

## 📋 内容规范

### 文件命名

- 使用小写字母和连字符：`01-rubber-band.md`
- 文件名应具备自解释性

### Commit Message 规范

请使用以下格式：

```
<type>: <description>

类型 (type):
- feat: 新功能或新内容
- fix: 修复错误
- docs: 文档更新
- style: 格式调整（不影响代码逻辑）
- refactor: 重构

示例:
feat: 添加橡皮筋战术实战案例
fix: 修正 GEX 计算公式错误
docs: 更新资源链接
```

### 写作风格

- **简洁明了**：避免过于学术化的表达
- **实战导向**：内容应该对交易有实际帮助
- **中文优先**：主要面向中文用户
- **适当英文**：专业术语可保留英文
